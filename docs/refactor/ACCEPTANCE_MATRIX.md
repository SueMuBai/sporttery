1. 从旧报错状态直接进入“数据更新”，确认不会再出现 `BeginTransaction: Already in transaction`。
2. 同步过程中切到后台、锁屏、强制结束后重新打开，再次同步确认残留事务可恢复。
3. 在方案改名、备注、回款输入时弹出输入法，确认底部主按钮可见并可点击。
4. 分别在一级页、二级页、底部 Sheet、确认框中按硬件返回键，确认每次只关闭最上层交互。
5. 冷启动、数据持久化、导出分享、刷新失败后本地数据继续可用。
