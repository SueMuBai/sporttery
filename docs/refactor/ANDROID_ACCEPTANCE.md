## 1. 构建前门禁

```bash
npm ci
npm test
npm run lint
npm run android:sync
npm run verify:runtime
```

必须满足：

- 自动测试全部通过。
- TypeScript、ESLint、Stylelint 无错误。
- Capacitor 发现 SQLite、App、Filesystem、Share 四项插件。
- Android 资源目录使用 Vite `dist`，不存在 Python、Chaquopy 或 localhost server。

## 2. GitHub Actions 构建

手动运行 `Build signed Android APK`：

- `version_name`：当前候选使用 `2.0.2`。
- `version_code`：必须大于已安装版本，当前候选至少使用 `4`。

保存以下证据：

- Workflow Run URL 和 Run ID。
- 所有步骤的绿色状态。
- `app-release.apk`。
- `app-release.apk.sha256`。
- `build-info.txt` 中的提交 SHA、`web_entry` 和 `web_entry_sha256`。
- APK 内对应 `web_entry` 的 SHA-256 必须与 `dist` 完全一致。
- APK 文件大小和 SHA-256 校验值。

## 3. 新安装验收

在没有安装旧版本的设备执行：

1. 安装签名 APK。
2. 首次启动不联网。
3. 确认账单、选票、设置三个主导航可进入。
4. 本地空数据状态正确，不出现白屏或 localhost 连接失败。
5. 恢复网络，执行一次数据更新。
6. 选择两场比赛和过关方式，保存方案；再单独执行“记录购买”。
7. 彻底结束 App 后重新启动，确认比赛、方案和账单仍存在。

## 4. 数据持久化与重装边界

当前版本不兼容旧数据，只验收全新数据库行为：

1. 全新安装后创建标签、方案和两笔购买账单。
2. 强制结束 App 并重新启动，确认数量、方案快照和金额不变。
3. 在同步写入过程中切到后台，再返回并重试，数据库不得卡在事务中。
4. 模拟残留事务后执行购买或方案导入：恢复成功时必须整批提交；恢复失败时数据库数量必须完全不变，不允许出现只有方案没有账单等半完成状态。
5. 覆盖安装同一新架构版本，确认当前数据库仍可正常读取。
6. 卸载后重新安装应得到空数据库，界面明确展示空状态。

## 5. 生命周期与系统交互

- Popup 打开时按返回键：只关闭 Popup。
- Dialog 打开时按返回键：执行取消，不确认危险操作。
- 二级设置、方案详情按返回键：返回上一级。
- 通过通知、调试链接或恢复状态直接打开二级路由且没有 WebView 历史时，返回键进入稳定父页面，不停留原页也不退出 App。
- 连续快速按两次返回键只处理一次最上层动作，不同时关闭浮层并退出页面。
- 账单、选票、设置根页面按返回键：退出 App。
- 切到后台 30 秒再恢复：当前选票草稿、滚动位置和编辑内容不丢失。
- 旋转屏幕或触发配置变化：不白屏、不重复初始化或并发开启事务。
- Activity 使用 `adjustResize`；键盘弹出后 WebView 可视高度实际缩小。
- 在方案改名、账单备注、实际回款和记录购买备注中分别弹出键盘，固定操作栏或 Sheet footer 完全位于键盘上方。
- 键盘弹出时打开和关闭底部面板：输入框及按钮不被遮挡；关闭键盘后页面高度和滚动位置恢复。

## 6. 离线、弱网和恢复

- 离线冷启动能够读取本地数据。
- 离线点击刷新显示失败提示，但本地比赛不消失。
- 同步中断后恢复网络并重试，已成功保存的数据不重复。
- 历史请求部分失败时显示部分失败数量，比赛列表仍可操作。
- 重复快速点击刷新只执行一个同步任务。

## 7. 文件导出

- JSON 导出能调起系统分享/保存面板。
- Markdown 导出能调起系统分享/保存面板。
- 保存后文件名、扩展名和内容正确。
- JSON 能被重新解析，数量与 App 页面一致。
- 取消系统分享不导致 App 崩溃或卡在 Loading。
- “导入方案备份”能调起 Android 文件选择器并仅接受 JSON 文件。
- 导入前显示方案、标签、引用比赛和最新赛果的准确数量与覆盖规则；取消确认不修改数据库。
- 导入成功后同 ID 方案按备份覆盖，标签及引用比赛/赛果同步恢复，本地更新的比赛/赛果不降级，已有账单和冻结快照保持不变。
- 导入覆盖正在其他页面编辑的同 ID 方案后，旧编辑页保存必须提示版本冲突，不得覆盖导入内容。
- 导入文件损坏、版本不支持、引用缺失标签或缺少方案引用的比赛快照时整体失败，不留下半导入方案。
- 超过标签/方案限制、同场多玩法、重复选项、无效过关、倍数或赔率的文件必须在确认前提示错误，数据库数量保持不变。

## 8. 界面与设备尺寸

至少覆盖等效宽度：

- 360 × 800。
- 390 × 844。
- 412 × 915。

每档保存账单、选票五种玩法、设置、日期筛选、方案详情截图。检查：

- 状态栏不遮挡标题。
- 底部系统导航不遮挡投注栏或底部导航。
- 无横向滚动。
- 所有边框和图标可见。
- 按钮文字水平和垂直居中。
- 系统字体 115% 时不截断关键金额和操作按钮。

## 9. 失败回滚

出现任何阻断问题时：

1. 不发布 2.0 APK。
2. 保留失败 Run 和日志。
3. 保留当前测试数据库和诊断日志。
4. 回装上一个新架构稳定 APK。
5. 记录问题设备、Android 版本、操作步骤、截图和数据库数量差异。
