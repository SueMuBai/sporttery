# 彩果 2.0 升级说明

## 架构变化

彩果 2.0 已从“Android 内运行 Python 本地 HTTP 服务”升级为原生 Capacitor 应用架构：

- 页面由 Vue 3 + TypeScript 构建并直接打包进 App。
- Android 数据使用 Capacitor Community SQLite。
- 不再包含 Chaquopy、Python 解释器或 localhost 服务。
- 启动链路更短，离线启动不依赖本地端口。

## 数据迁移

从旧版本覆盖安装时：

- App 会查找旧版 `filesDir/caiguo.db`。
- 只有在新数据库不存在时才复制旧数据库。
- 浮点“元”金额会迁移为整数“分”，避免金额累计误差。
- 旧数据库不会被删除或覆盖，可用于回滚和核对。
- 迁移重复执行不会重复创建方案或账单。

不要在升级前卸载旧 App。卸载可能同时清除应用私有目录中的数据库。

## 升级前建议

1. 在旧版中确认方案和账单能够正常打开。
2. 记录方案数、账单数、总投入和总回款。
3. 保留旧版 APK。
4. 直接覆盖安装 2.0 APK。
5. 第一次打开后先核对数据，再进行同步。

## 2.0 新能力

- 五种玩法统一选票界面。
- 真实方案管理、导入编辑和冲突处理。
- 长期账单、日期筛选和回款修正。
- 当前收益、完成场次和猜对数量。
- 标签管理和系统同步参数。
- JSON/Markdown 文件导出。
- 统一晴空汽水主题、边框、图标和移动端组件。

## 回滚

如 2.0 首次启动后发现数据数量或金额不一致：

- 不要继续编辑或同步数据。
- 不要卸载 App。
- 保存截图和错误信息。
- 使用旧稳定 APK 覆盖回装。
- 旧 `caiguo.db` 仍应保留，可用于重新迁移。
