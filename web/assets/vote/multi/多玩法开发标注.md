# 多玩法开发标注

## 比分

- 固定 5 列，间距 8dp；单元格文字两行，上行为比分/其它，下行为赔率。
- 不显示“主胜方案/平方案/客胜方案”等分组标题。
- 数据中的空位置仅用于保持排列顺序，不渲染空按钮。

## 总进球

- 0球至7+球采用 4×2 网格，无横向滚动。

## 半全场

- 使用 3×3 矩阵；列表示全场结果，行表示半场结果。
- 读屏标签建议使用“半场胜、全场平，赔率14.00”这类完整描述。

## 混合过关

- 每场比赛内按 `胜平负/让球 → 比分 → 总进球 → 半全场` 排列。
- 同一场一次只展开一个玩法，切换时自动收起原玩法。
- 收起状态显示选项数量、已选数量和最多一个已选摘要；完整已选项在底部方案面板查看。
- 展开比分时直接复用独立比分页的 5 列布局。

## 通用交互

- 选项最小点击高度 44dp。
- 折叠动画建议 240ms ease-out。
- 页面底部选择栏固定在导航上方，默认折叠。
