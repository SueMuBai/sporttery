# 设置模块开发标注

## 信息架构

- 一级设置页仅承担导航，不直接显示可编辑输入框或标签编辑控件。
- 分组标题位于白色列表卡片外部；卡片内部使用行分隔线。
- `系统设置`、`标签管理` 等项目使用 push 导航进入完整二级页面。
- 二级页面隐藏底部导航，左上角固定返回按钮；同时支持系统返回手势/返回键。

## 系统设置

- 参数使用步进器；减号、数值、加号分别为独立点击区域。
- 每场历史条数：10；并发请求数：4；接口超时：15秒；失败重试次数：2；默认倍数：1。
- 设计不硬编码最大值，实际上下限由业务配置；默认倍数不得小于1。
- 修改后点击“保存设置”统一提交。
- 存在未保存修改时返回，应提示“保存修改 / 放弃修改 / 继续编辑”。

## 标签管理

- 标签列表显示名称、用途说明、编辑和删除操作。
- 标签名称最多12个字符；同一名称建议忽略大小写后保持唯一。
- 删除标签仅移除标签关联，不删除方案。
- 同一方案允许添加多个标签。
- 点击“新增标签”进入新增流程；成功后返回列表并滚动到新标签。

## 导出

- Markdown和JSON入口位于一级设置目录。
- 导出完成后展示文件名、保存位置或分享入口；失败时保留重试操作。

## 无障碍

- 行和按钮最小点击高度44dp。
- 步进器需提供完整读屏标签，例如“并发请求数，当前4，减少/增加”。
