# 晴空汽水主题修改方案

## 目标

将原有大面积墨绿色主题调整为清新、年轻、轻二次元的“晴空汽水”风格。保留现有信息架构和交互，只替换主题变量、选中态皮肤、阴影和少量页头装饰。

## 修改原则

- 彩色面积控制在页面约15%–20%，卡片和主要内容仍以白色为主。
- 不增加动漫人物、复杂插画或高饱和霓虹效果。
- 胜/盈利、平/提醒、负/亏损仍使用独立语义色，不被品牌色覆盖。
- 所有文字和动态数据继续由客户端原生渲染，不烘焙到切图。

## 页面级修改

### 选票与多玩法

- 页头统一替换为晴空蓝到淡紫渐变，并使用少量气泡和星点装饰。
- 当前玩法使用樱花粉；已选赔率使用樱花粉到柔珊瑚渐变。
- 过关选中项使用蓝紫渐变；查看方案为蓝紫主按钮。
- 比分、总进球、半全场和混合过关只更换颜色变量，不调整已确认布局。

### 账单

- 标题栏使用同一晴空渐变。
- 总回款/正收益使用薄荷绿，亏损使用柔珊瑚红。
- 日期区间使用极浅蓝紫范围底色，端点使用晴空蓝。
- 进行中状态使用淡柠檬黄；已完成状态使用淡薄荷绿。

### 设置

- 一级目录图标按蓝、紫、薄荷分组，但同一列表不超过三种强调色。
- 二级页面返回按钮保持半透明白底；步进器焦点和保存按钮使用晴空蓝。
- 标签颜色可使用薄荷绿、汽水蓝、樱花粉和淡紫。

## 实施顺序

1. 新增主题颜色变量，保留旧变量一版作为回滚。
2. 替换页面背景、页头、文字、边框和阴影等基础Token。
3. 替换按钮、玩法标签、赔率、过关选项等交互状态。
4. 替换账单正负金额、日期范围和状态标签。
5. 替换设置图标和标签颜色。
6. 对全部页面执行对比度、状态完整性和暗色文字可读性检查。

## 不需要重新切图的内容

- 搜索、刷新、箭头、底部导航等线性图标可直接通过Tint换色。
- 白色卡片、描边按钮、输入框、赔率格和标签由原生组件/CSS绘制。
- 只有页头装饰、蓝紫渐变按钮和特殊选中态需要使用本包主题切图，或按SVG/CSS等价实现。
