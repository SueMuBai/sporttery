# Compact V2 UI 资源

运行时 UI 资源统一保存在本目录，`img/` 仅作为设计源和验收参考，业务代码不得直接引用 `img/`。

当前视觉基准：

`img/ui/彩果_晴空汽水紧凑版V2_完整补充交付包`

机器可读接入状态见 [`manifest.json`](./manifest.json)。设计包只用于开发期核对，构建产物不依赖该目录。

目录约定：

- `common/`：一级标头等跨模块资源。
- `ticket/`：选票、玩法、赔率和投注栏资源。
- `ledger/`：账单、日期入口和详情 Sheet 资源。
- `settings/`：设置列表、步进器和设置业务图标。
- `date/`：年月日选择器资源。
- `plans/`：方案管理、方案详情和标签管理资源。
- `theme/`：旧主题兼容资源；新增代码优先使用上述语义目录。
- 底部导航位于 `src/assets/icons/navigation/`，六个默认/选中图标与最新补充包一致。

接入规则：

1. 只有实际被页面使用的资源才迁入项目。
2. 卡片、按钮、输入框、描边、分割线和 Sheet 背景优先由 CSS 绘制。
3. SVG 通过 `?url` 或图标组件引用，禁止运行时拼接 `img/` 路径。
4. 新资源按模块命名；跨模块复用资源放入 `common/`，避免重复副本。
5. 外链 SVG 需要动态着色时使用 `AppAssetIcon` 的 CSS mask；不要使用 CSS filter 强行改色。
6. 从旧交付包迁入的线性图标统一把写死的绿色 stroke 规范化为 `currentColor`。

当前接入摘要：

- 设置一级页、系统设置、数据与备份、数据更新和关于页已使用迁入 SVG。
- 选票和账单缺失图标已经放入各自语义目录，可由对应页面直接接入。
- `date/`、`plans/` 已在此前迁移；底部导航六个 SVG 与补充包逐文件哈希一致。
- 新补充包中的 `bg_*` 控件背景未再次迁移；边框、卡片、按钮和状态背景继续由 CSS 绘制。
